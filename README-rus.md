# MMSS - Modular Motion Sound System

Сложная система модульного синтезатора и визуального перформанса на базе React, которая сочетает генеративный аудиосинтез, динамическую визуализацию и интерактивные элементы управления для создания иммерсивных мультимедийных впечатлений.

## Обзор

MMSS (Modular Motion Sound System) - это продвинутое веб-приложение, которое интегрирует несколько систем для реального времени аудио-визуального перформанса. Система включает матричный секвенсор, орбитальное управление движением, анализ изображений и обработку естественного языка для создания эволюционирующих звуковых ландшафтов и визуалов.

## Ключевые возможности

### 🎵 Аудиодвижок
- **Интеграция Web Audio API**: Синтез в реальном времени с использованием осцилляторов, модуляции и эффектов
- **Матричный секвенсор**: Сетка 24x12 для композиции музыки на основе паттернов
- **Динамическая генерация голосов**: Голоса с множественными осцилляторами и частотной модуляцией
- **Реал-тайм транспорт**: Точное управление темпом и синхронизацией

### 🎨 Визуальная система
- **Рендеринг на Canvas**: Высокопроизводительные визуальные эффекты с градиентными фонами
- **Визуализация аудио**: Осциллограф и отображение волновых форм в реальном времени
- **Орбитальная анимация**: Плавные переходы между различными визуальными конфигурациями
- **Система тем**: Динамические цветовые палитры и визуальные параметры

### 🎛️ Системы управления
- **Матричный редактор**: Интерактивное создание и редактирование паттернов
- **Орбитальное движение**: Автоматизированная интерполяция параметров и рандомизация на основе коллизий
- **Анализ изображений**: Компьютерное зрение для извлечения аудиопараметров из изображений
- **Обработка текстовых намерений**: Команды на естественном языке для управления системой

### 🔄 Управление состоянием
- **Прогрессивные уровни**: Система эволюционирует через уровни сложности L0-L4
- **Система чекпоинтов**: Сохранение состояния и отслеживание прогресса
- **Архитектура редьюсера**: Централизованное управление состоянием с React хуками

## Архитектура

Приложение структурировано вокруг 8 взаимосвязанных систем:

### 1. Система инициализации
- Загрузка React приложения
- Настройка аудиоконтекста
- Запуск и инициализация основной системы

### 2. Управление состоянием
- Центральный `mmssReducer` для всех изменений состояния
- Прогрессивная система уровней с чекпоинтами
- Управление состоянием транспорта и аудиопараметрами

### 3. Аудиодвижок (`useAudioEngine`)
- Управление контекстом Web Audio API
- Транспортный цикл с точной синхронизацией
- Триггеринг голосов и планирование нот
- Создание осцилляторов с цепями модуляции

### 4. Визуальный рендеринг (`StageCanvas`)
- Двойная система Canvas (сцена + осциллограф)
- Градиентные фоны и кольцевые анимации
- Визуализация аудиоволн в реальном времени
- Управление цветами на основе тем

### 5. Матричный редактор
- Интерактивная сетка 24x12 для создания паттернов
- Поддержка касаний и мыши
- Визуализация плэйхэда в реальном времени
- Режимы рисования и стирания ячеек

### 6. Орбитальное движение (`useOrbitMotion`)
- Автоматизированная интерполяция параметров
- Обнаружение коллизий и рандомизация
- Смешивание состояний с функциями сглаживания
- Микширование аудио и визуальных параметров

### 7. Анализ изображений
- Загрузка и обработка файлов
- Извлечение данных изображений через Canvas
- Расчет контрастности, плотности краев и симметрии
- Отображение параметров на аудиоэлементы управления

### 8. Обработка намерений
- Парсинг команд на естественном языке
- Переключение сцен через ключевые слова
- Корректировка аудиопараметров
- Предустановленные конфигурации

## Использование

### Начало работы
1. Запустите приложение - система инициализируется автоматически
2. Аудиодвижок запускается при первом взаимодействии с элементами управления
3. Используйте матричный редактор для создания паттернов
4. Экспериментируйте с орбитальным движением для автоматизированной эволюции

### Элементы управления

#### Транспорт
- **Воспроизведение/Пауза**: Переключение воспроизведения последовательности
- **BPM**: Настройка темпа (влияет на время всех операций)

#### Матричный редактор
- **Клик/Перетаскивание**: Рисование или стирание ячеек в сетке 24x12
- **Цвета**: Каждая строка представляет разную высоту тона/ноту
- **Плэйхэд**: Визуальный индикатор текущей позиции последовательности

#### Орбитальное движение
- **Включить/Выключить**: Переключение автоматизированной интерполяции параметров
- **Скорость**: Управление скоростью орбитального движения
- **Коллизии**: Случайные изменения параметров на границах состояний

#### Анализ изображений
- **Загрузка изображения**: Перетащите или выберите файлы изображений
- **Автоанализ**: Извлекает контраст, края и симметрию
- **Отображение параметров**: Преобразует визуальные метрики в аудиоэлементы управления

#### Текстовые намерения
- **Ввод текста**: Команды на естественном языке, такие как "glass orbit", "aurora gate"
- **Переключение сцен**: Ключевые слова активируют предустановленные конфигурации
- **Корректировка параметров**: Слова типа "calm", "intense" изменяют настройки

### Горячие клавиши
- Пробел: Воспроизведение/Пауза транспорта
- Колесико мыши: Настройка параметров (в зависимости от контекста)

## Технические детали

### Аудиоархитектура
- **Частота дискретизации**: Зависит от браузера (обычно 44.1кГц)
- **Структура голоса**: Основной осциллятор + вторичный осциллятор + модуляция
- **Цепь эффектов**: Стадирование усиления с узлами Web Audio
- **Синхронизация**: Высокоточное планирование с циклами `setTimeout`

### Визуальная производительность
- **Частота кадров**: Цель 60fps с `requestAnimationFrame`
- **Разрешение Canvas**: Динамическое масштабирование под размер контейнера
- **Глубина цвета**: 32-битный RGBA для плавных градиентов
- **Управление памятью**: Эффективное повторное использование буферов и очистка

### Сохранение состояния
- **Локальное состояние**: React useReducer с неизменяемыми обновлениями
- **Прогрессия уровней**: Система продвижения с чекпоинтами
- **Диапазоны параметров**: Ограниченные значения для стабильной работы

## Совместимость с браузерами

- **Chrome/Edge**: Полная поддержка с Web Audio API
- **Firefox**: Поддерживается с незначительными различиями в синхронизации
- **Safari**: Ограниченная поддержка (особенности Web Audio API)
- **Мобильные**: Интерфейс оптимизирован для касаний
